# cargo mkdist – полная документация

## Введение

`cargo mkdist` - это подкомманда Cargo, которая собирает родные пакеты
дистрибьюции из проектов на Rust **вообще без вызова внешних упаковщиков**
по типу `dpkg-deb`, `rpmbuild` или `makepkg`.

mkdist читает Ваш манифест и простой конфиг в `.dist/targets.toml`, компилирует
Ваш бианрник для соответствующей целевой тройки, собирает файлы в "staging area"
и упаковывает их в требуемый формат, используя Rust-библиотеки.

В кратце, это самодостаточная упаковочная утилита, которая работает везде, где и Rust.

---

## Установка

```bash
cargo install cargo-mkdist
```

После установки, команда `cargo mkdist` становится доступной.

---

# Пользование CLI

```bash
cargo mkdist [OPTIONS] [TARGET] [-- <CARGO_BUILD_ARGS>...]
```

### Опции

| Флаг              | Описание                                                          |
| ----------------- | ----------------------------------------------------------------- |
| `--list`          | Перечислить все доступные из конфигурации цели.                   |
| `--all`           | Собрать все доступные из конфигурации цели.                       |
| `--debug`         | Собирать в отладочном режиме (по умолчанию сборка релизная)       |
| `--out-dir <DIR>` | Установить папку для выхлопа (по умолчанию `target/packages`).    |
| `--help`          | Показать справку.                                                 |
| `--version`       | Показать версию.                                                  |

если `TARGET` задан, только эта цель будет упакована.
Если цель не задана и `--all` не используется, будет выбрана цель `default` (если такая есть).

Все агрументы после `--` направляются прямо в `cargo build`.

### Примеры

```bash
# Собрать цель по умолчанию.
cargo mkdist

# Собрать конкретную цель.
cargo mkdist deb

# Собрать все цели.
cargo mkdist --all

# Собрать для pacman с фишкой foo.
cargo mkdist arch -- --features=foo

# Перечислить досутпные цели.
cargo mkdist --list
```

## Конфигурация

Конфигурация находится в `dist-targets.toml` либо `.distinfo/targets.toml`
(поиск в текущей рабочей директории, `.distinfo/` и `.cargo/`).

Файл должен содержать таблицу `[targets]`, где ключ - название цели, значение - поля цели.

### Поля цели

| Поле          | Тип               | Обязат. ? | Описание                                                                                          |
| ------------- | ----------------- | --------- | ------------------------------------------------------------------------------------------------- |
| `target`      | строка            | нет       | Тройка цели Rust (напр., `x86_64-unknown-linux-gnu`). Если опущено, обязано быть наследовано.     |
| `format`      | строка            | да        | Формат пакета: `deb`, `pkg`, `tgz`.                                                               |
| `inherit`     | строка            | нет       | Название цели для наследования из неё.                                                            |
| `binaries`    | список строк      | нет       | Какие бинарники включить в пакет. Если опущено, все `[[bin]]` цели из `Cargo.toml` использованы.  |
| `package`     | строка            | нет       | Название пакета текущего рабочего пространства для сборки (если не корневой пакет).               |
| `flags`       | строка            | нет       | Дополнительные флаги для передачи в `cargo build` (напр., `--features=foo`).                      |
| `extra`       | таблица           | нет       | Формат-специфичные флаги (см. далее).                                                             |

Все поля можут наследоваться из родительской цели. Наследованные значения переназначаются явными значениями в наследниках.

### Пример наследования

```toml
# Базовая цель
[targets.linux-base]
target = "x86_64-unknown-linux-gnu"
flags = "--release"
binaries = ["myapp"]

# Наследник 1
[targets.deb]
inherit = "linux-base"
format = "deb"
extra = { section = "utils" }

# Наследник 2
[targets.arch]
inherit = "linux-base"
format = "pkg"
extra = { license = "MIT" }
```

Здесь и `deb`, и `arch` наследуют `target`, `flags` и `binaries` от `linux-base`.

## Формат-специфичные флаги

### `deb` – пакет Debian

В `extra` могут быть такие поля:

| Поле                                  | Тип       | Описание                                                                          |
| ------------------------------------- | --------- | --------------------------------------------------------------------------------- |
| `section`                             | строка    | например, `utils`, `admin` или `net` (по умолчанию `utils`).                      |
| `priority`                            | строка    | `optional`, `required`, `important` или `standard` (по умолчанию `optional`)      |
| `maintainer`                          | строка    | Мейнтейнер (по умолчанию из поля `authors` в манифесте пакета).                   |
| `description`                         | строка    | Описание (по умолчанию из манифеста пакета)                                       |
| `depends`                             | строка    | Разделённый запятой список зависимостей, наприм., `libc6 (>= 2.34), libssl3`.     |
| `recommends`                          | строка    | Рекомендованные пакеты, разделённые запятой.                                      |
| `suggests`                            | строка    | Предложенные пакеты, разделённые запятой.                                         |
| `conflicts`                           | строка    | Конфликтующие пакеты, разделённые запятой.                                        |
| `breaks`                              | строка    | Ломающие версию пакеты, разделённые запятой.                                      |
| `replaces`                            | строка    | Разделённый запятой список пакетов, которые заменяются Вашим.                     |
| `provides`                            | строка    | Разделённый запятой список виртуальных пакетов, предоставляемых Вашим пакетом.    |
| `enhances`                            | строка    | Разделённый запятой список пакетов, которые улучшены Вашим.                       |
| `pre-depends`                         | строка    | Разделённый запятой список пред-зависимостей.                                     |
| `conffiles`                           | строка    | Разделённый запятой список файлов конфигурации (напр., `/etc/myapp/config.toml`)  |
| `preinst`,`postinst`,`prerm`,`postrm` | строка    | Пути к скриптам мейнтейна (относительно корня проекта).                           |

---

### `pkg` – пакет Arch Linux (`.pkg.tar.zst`)

| Поле          | Тип       | Описание                                                              |
| ------------- | --------- | --------------------------------------------------------------------- |
| `pkgrel`      | число     | Номер релиза пакета (по умолчанию 1)                                  |
| `epoch`       | число     | Эпоха (опционально)                                                   |
| `license`     | строка    | Строка лицензии (наприм., `MIT`)                                      |
| `url`         | строка    | Upstream URL                                                          |
| `depends`     | строка    | Разделённый запятой список зависимостей.                              |
| `optdepends`  | строка    | Разделённый запятой список опциональных зависимостей.                 |
| `conflicts`   | строка    | Разделённый запятой список пакетов, конфликтующих с Вашим.            |
| `provides`    | строка    | Разделённый запятой список предоставленных виртуальных пакетов.       |
| `replaces`    | строка    | Разделённый запятой список заменённых пакетов.                        |
| `backup`      | строка    | Разделённый запятой список файлов конфигурации (напр. `/etc/my.ini`). |
| `install`     | строка    | Путь к скрипту `.INSTALL` (относительно корня проекта).               |

---

### `tgz` – simple tarball

Никаких дополнительных полей. Вся папка стейжинга просто упаковывается в tgz-архив.

## Как это работает (изнутри)

1. **Чтение конфигурации** – загружает `targets.toml` и разрешает наследования.
2. **Сборка** – запускает `cargo build --target <target>` с выданными ффлагами.
3. **Подготовка стейжинга** – создаёт временную папку.
4. **Копирование исполняеых файлов** – копирует собранные исполняемые файлы в стейжинг.
5. **Доп-файлы** – копирует в стейжинг доп-файлы (если настроено).
6. **Эмиссия метаданных** – генерирует метаданные пакета в соответствии с его форматом.
7. **Упаковка** – производит финальный пакет:
   - `deb`: результат - `*.deb`-пакет, готовый к установке.
   - `pkg`: результат - `.pkg.tar.zst`-пакет, готовый к установке.
   - `tgz`: результат - простой архив стейжинга.
8. **Вывод** – сохраняет пакет в `target/packages/<target-name>/`.

## Примеры

### Минимальный конфиг для единственного бинарника

```toml
[targets.default]
target = "x86_64-unknown-linux-gnu"
format = "deb"
binaries = ["myapp"]
extra = {
    depends = "libc6 (>= 2.34)",
    section = "utils"
}
```

### Несколько целей с наследованием

```toml
[targets.base]
target = "x86_64-unknown-linux-gnu"
flags = "--release"
binaries = ["myapp"]

[targets.deb]
inherit = "base"
format = "deb"
extra = {
    depends = "libc6 (>= 2.34), libssl3",
    conffiles = "/etc/myapp/config.toml",
    postinst = "scripts/postinst.sh"
}

[targets.arch]
inherit = "base"
format = "pkg"
extra = {
    pkgrel = 1,
    depends = "glibc, openssl",
    license = "MIT",
    install = "scripts/arch-install.sh"
}
```

### Кросс-компиляция

```toml
[targets.arm-deb]
target = "aarch64-unknown-linux-gnu"
format = "deb"
binaries = ["myapp"]
extra = {
    depends = "libc6 (>= 2.34)"
}
```

У Вас должна быть установлена соответсвующая Rust-цель (`rustup target add aarch64-unknown-linux-gnu`) и кросс-компилятор должен быть доступен.

## Ограничения / Замечания / Известные недостатки

- Пока что поддерживаются только `deb`, `pkg.tar.zst` и `tgz`. Как минимум RPM и MSI уже запланированы.
- Права сохраняются на UNIX (режимы файлов, ссылки). Поддержка Windows для упаковки ещё не протестирована тщательно.
- Скрипты и конффайлы поддерживаются только для `deb`; `pkg` поддерживает `.INSTALL`.
